## ğŸ”§ Troubleshooting

### Issue: Backend won't start

**Problem:** `ModuleNotFoundError` or import errors

**Solution:**
- Make sure virtual environment is activated (you should see `(venv)` in your terminal prompt)
- Reinstall dependencies:
  ```bash
  # Windows
  pip install -r requirements.txt
  pip install 'pydantic[email]'
  
  # Mac/Linux
  pip install -r requirements.txt
  pip install 'pydantic[email]'
  ```

**Problem:** Port 8000 already in use

**Solution:**
- Find and stop the process using port 8000:
  ```bash
  # Windows
  netstat -ano | findstr :8000
  taskkill /PID <PID> /F
  
  # Mac/Linux
  lsof -ti:8000 | xargs kill -9
  ```
- Or change the port in `backend/main.py` (line 60)

---

### Issue: Frontend won't start

**Problem:** `npm install` fails

**Solution:**
- Clear npm cache:
  ```bash
  npm cache clean --force
  ```
- Delete `node_modules` and reinstall:
  ```bash
  # Windows
  rmdir /s node_modules
  npm install
  
  # Mac/Linux
  rm -rf node_modules
  npm install
  ```

**Problem:** Port already in use

**Solution:**
- Vite will automatically try another port (check the terminal output)
- Or kill the process using the port:
  ```bash
  # Windows
  netstat -ano | findstr :5173
  taskkill /PID <PID> /F
  
  # Mac/Linux
  lsof -ti:5173 | xargs kill -9
  ```

---

### Issue: Registration/Sign Up not working

**Problem:** CORS errors or "Registration failed"

**Solution:**
- Make sure backend is running on port 8000
- Check that the frontend port is included in backend CORS settings (`backend/main.py`)
- Check browser console (F12) for specific error messages
- Verify backend logs show the request being received

---

### Issue: Database errors

**Problem:** `cybersecure.db` not found or corrupted

**Solution:**
- Delete the database file and restart backend (it will recreate automatically):
  ```bash
  # Windows
  del backend\cybersecure.db
  
  # Mac/Linux
  rm backend/cybersecure.db
  ```
- Restart the backend server

## ğŸ§  Key API Endpoints

| Method | Endpoint            | Description                               |
| ------ | ------------------- | ----------------------------------------- |
| `POST` | `/register`         | Register a new user                       |
| `POST` | `/login`            | Authenticate user and receive JWT         |
| `GET`  | `/articles`         | Fetch all articles                        |
| `GET`  | `/articles/{id}`    | Fetch full article by ID                  |
| `GET`  | `/quizzes`          | Get a random quiz                         |
| `POST` | `/submit_quiz`      | Submit quiz results (Auth required)       |
| `GET`  | `/scores/{user_id}` | Fetch user's quiz history (Auth required) |

**API Documentation:** Visit `http://localhost:8000/docs` when backend is running for interactive API docs.

## ğŸ”’ Authentication Flow

1. On successful login, backend returns a **JWT**:
   ```json
   { "token": "<jwt_token>" }
   ```
2. Frontend stores it in `localStorage` under `auth_token`.
3. Protected routes automatically attach:
   ```
   Authorization: Bearer <token>
   ```
4. If token is expired or invalid â†’ frontend logs user out.
